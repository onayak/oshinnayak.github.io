Creating User Archetypes

This project’s end result was changing the project itself.

The design team had conducted interviews with various customers like infrastructure admins, network admins, and project managers. The end goal of the project at that stage was to create personas to help designers and stakeholders.

I took the task of analysing these interviews using Nvivo to code the transcripts.

Overall my analysis led to one major insight: The interview script needs to be modified.

In the infrasture world role or position of users and their work aren’t mapped consistently through the university. This led us to also consider modifying the interview scripts and the study plan itself. Instead of just interviewing customers we decided to talk to employees who had been in the firm for years and could shed light on issues customers faced in detail. We also updated the end goal to Creating User Archetypes, instead of just personas.

Redesigning File Servers

To create high fidelity prototypes for Nutanix Prism components was the most challenging part of my internship. Specifically because I had to redesign a perfectly functional and usable component.

Well then why redesign at all?

Through usability tests with customers it had been clear that most users did not know much about how the process of creating a file sever works on Nutanix Node. With no direct competitors this wasn't a conventional workflow and was specific to Nutanix Prism.

Goal: To help users have a clear understanding of the File server creation process so that they cab efficiently utilize the available resources.

How did I design a new flow with visual aspects?

  • Analysed the usability data from .NEXT Nice and DC for Visual flows for AFS.
  • Used maze.design to share the redesigned prototypes and collect comprehensive feedback to use for future iterations.
  • Conducted usability tests with UI engineer and a backend engineer.

Visualize your Network

In a three day Hackathon my team and I tried to answer a common question Network admins have: Which node is not working? In the world of datacenter admins we learned that error dectection is a complex time consuming task.

Mapping physical connections in a datacenter to a visual network topology would help reduce some of this complexity. Our proposed design used simple shapes (nodes) and red colored dots (errored link source) to indicate overall state of the datacenter. Our deign rationale was based on the following

To monitor a complex system where mutiple things can go wrong, an efficient error dectection system could be static when all systems are up. In case of an error, it could indicate it through motion, color, and concentrated activity in the errored region.
Reduce the cognitive load by displaying details in progressive manner.

Due to the NDA agreement the designs cannot be shared here.
